## 📌 Overview

The `/firmware/edge_impulse` directory contains the exported C++ library generated by the **Edge Impulse** platform. This represents the absolute "lowest" tier of AI in the AyushBot ecosystem: an embedded neural network running directly on the constrained ESP32 microcontroller inside the patient sensor pack.

## 🎯 Purpose: Motion Artifact Rejection

Pulse Oximeters (MAX30102) are notoriously susceptible to motion artifacts. If a crying child moves their arm, the raw photoplethysmogram (PPG) wave distorts, causing the sensor to falsely report an SpO2 of 40%.

Instead of sending this garbage data over BLE to the Android tablet (which would trigger a false CRITICAL alarm and panic the ASHA), the sensor pack runs a tiny 1D Convolutional Neural Network against the raw I2C buffer.

## 🧩 Integration

This directory is not modified by hand. It is dropped in as an exported `.zip` from the Edge Impulse Studio. 
- The primary entry point used by `/firmware/sensor_pack/src/main.cpp` is `#include "edge-impulse-sdk/classifier/ei_run_classifier.h"`.
- Total inference time per 2-second window is ~5ms on an ESP32 at 240MHz.
